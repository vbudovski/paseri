It's possible to define a recursive schema using lazy evaluation. Unfortunately, due to a limitation of TypeScript, its
type cannot be automatically inferred.

:::note
`safeParse` and `parse` accept a `maxDepth` option that caps the nesting depth of your recursive data. The default of
`1000` is generous and you'll rarely need to change it. To accept deeper input from a source you trust, override per
call: `schema.safeParse(data, { maxDepth: 5000 })`.
:::

## Example

```typescript
import * as p from '@paseri/paseri';

type Node =
    | { type: 'file'; name: string }
    | { type: 'directory'; name: string; nodes: Node[] };

// The explicit type annotation is required for recursive schemas.
const schema: p.Schema<Node> = p.lazy(() =>
    p.union(
        p.object({
            type: p.literal('file'),
            name: p.string(),
        }),
        p.object({
            type: p.literal('directory'),
            name: p.string(),
            nodes: p.array(schema),
        }),
    ),
);
const data: Node = {
    type: 'directory',
    name: 'Documents',
    nodes: [
        {
            type: 'file',
            name: 'shopping_list.txt',
        },
        {
            type: 'directory',
            name: 'Pictures',
            nodes: [
                { type: 'file', name: 'cat.jpg' },
                { type: 'file', name: 'dog.jpg' },
            ],
        },
    ],
};

const result = schema.safeParse(data);
if (result.ok) {
    // result.value typed as `Node`.
}
```
