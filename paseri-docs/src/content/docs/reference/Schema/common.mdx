All schemas contain some common methods.

## Methods

### `parse`

The `parse` method takes a `value` parameter as input. If the `value` conforms to the schema, then the parsed value is
returned with the appropriate type, otherwise an exception containing a `messages` method is thrown. See the
[messages](/reference/messages/) reference for details.

```typescript
import * as p from '@paseri/paseri';

const schema = p.string();
try {
    const parsedValue = schema.parse('foo');
    // Do something with the `string`.
} catch (e) {
    // Do something with the exception.
}
```

### `safeParse`

The `safeParse` method takes a `value` parameter as input. The result is a discriminated union with a `boolean` member
`ok`. You can use it to extract the parsed value on success or get the error [messages](/reference/messages/) using
`result.messages(locale)`. Pass a locale (e.g. `en`) to get human-readable strings; calling without a locale returns
the raw issue codes.

```typescript
import * as p from '@paseri/paseri';
import { en } from '@paseri/paseri/locales';

const schema = p.string();
const result = schema.safeParse('foo');
if (result.ok) {
    // Do something with the `result.value`.
} else {
    // Do something with the `result.messages(en)`.
}
```

### `optional`

Allow any schema to accept `undefined` as a valid value.

```typescript
import * as p from '@paseri/paseri';

const schema = p.string().optional();
const result = schema.safeParse(undefined);
if (result.ok) {
    // result.value is of type `string | undefined`.
}
```

#### `default`

To substitute a value when the input is `undefined`, follow `.optional()` with `.default(value)`.

```typescript
import * as p from '@paseri/paseri';

const schema = p.number().optional().default(123);
const result = schema.safeParse(undefined);
if (result.ok) {
    // result.value is of type `number`, equal to 123.
}
```

:::caution
Treat default values as immutable! Plain objects and arrays are frozen &mdash; mutations throw `TypeError`. Other
types (e.g. `Date`, `Map`, `Set`, class instances) have mutator methods that bypass the freeze, and mutations will
leak across parses sharing the default.
:::

### `nullable`

Allow any schema to accept `null` as a valid value.

```typescript
import * as p from '@paseri/paseri';

const schema = p.string().nullable();
const result = schema.safeParse(null);
if (result.ok) {
    // result.value is of type `string | null`.
}
```

### `chain`

Chain schemas to convert data types. The chain method takes the target schema as the first argument, and a transformer
function as the second argument.

:::caution
The transformer function must *not* throw! Instead, return an error code using the `err` helper function.
:::

```typescript
import * as p from '@paseri/paseri';

const schema = p.string().chain(p.bigint(), (value) => {
    try {
        return p.ok(BigInt(value));
    } catch {
        return p.err('invalid_bigint');
    }
});
// Number.MAX_SAFE_INTEGER + 1 as a string.
const result = schema.safeParse('9007199254740992');
if (result.ok) {
    // result.value is of type `bigint`.
}
```

:::note
The error code `'invalid_bigint'` is a custom code. Custom error codes require a
[custom messages object](/reference/messages/#custom-error-codes) to work with `result.messages()`.
:::

`err` accepts an optional second argument, `params`, for forwarding placeholder values to the
[matching locale entry](/reference/messages/#custom-error-codes).

```typescript
import * as p from '@paseri/paseri';

const schema = p.string().chain(p.number(), (value) => {
    const n = Number(value);
    if (Number.isNaN(n)) {
        return p.err('not_a_number', { input: value });
    }
    return p.ok(n);
});
```

### `refine`

Add a custom check to any schema. The predicate runs on the parsed value and if it fails, an error is emitted at the
given path. Multiple `.refine()` calls short-circuit on the first failure.

```typescript
import * as p from '@paseri/paseri';

const schema = p
    .object({
        password: p.string(),
        confirm: p.string(),
    })
    .refine((value) => value.password === value.confirm, {
        code: 'password_mismatch',
        path: ['confirm'],
    });
```

The `code` is required. `path` defaults to the root (`[]`) and accepts `string` or `number` segments. The optional
`params` object supplies placeholder values to the [matching locale entry](/reference/messages/#custom-error-codes).

:::caution
The predicate function must *not* throw! Catch exceptions inside the predicate and return `false` instead.
:::

## Types

### `Infer`

You can use the `Infer` type to determine the result type of any schema. This is useful, because you don't want to
manually define a type that could easily get out of sync with the schema definition.

```typescript
import * as p from '@paseri/paseri';

const schema = p.object({
    foo: p.string(),
    bar: p.number(),
})

type MySchema = p.Infer<typeof schema>;
// MySchema will be { foo: string, bar: number }.
```

## Standard Schema

Paseri also provides an interface that's compatible with [Standard Schema](https://standardschema.dev/schema). This
makes it possible to use Paseri with a number of tools and frameworks without having to write custom wrappers.
