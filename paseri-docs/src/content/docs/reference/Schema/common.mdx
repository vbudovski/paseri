All schemas share some common methods, and can be wrapped or composed with the helpers below.

## Methods

### `parse`

The `parse` method takes a `value` parameter as input. If the `value` conforms to the schema, then the parsed value is
returned with the appropriate type, otherwise an exception containing a `messages` method is thrown. See the
[messages](/reference/messages/) reference for details.

```typescript
import * as p from '@vbudovski/paseri';

const schema = p.string();
try {
    const parsedValue =
        schema.parse('foo');
    // Do something with the `string`.
} catch (e) {
    // Do something with the exception.
}
```

### `safeParse`

The `safeParse` method takes a `value` parameter as input. The result is a discriminated union with a `boolean` member
`ok`. You can use it to extract the parsed value on success or get the error [messages](/reference/messages/) using
`result.messages(locale)`. Pass a locale (e.g. `en`) to get human-readable strings; calling without a locale returns
the raw issue codes.

```typescript
import * as p from '@vbudovski/paseri';
import { en } from '@vbudovski/paseri/locales';

const schema = p.string();
const result = schema.safeParse('foo');
if (result.ok) {
    // Do something with the `result.value`.
} else {
    // Do something with the `result.messages(en)`.
}
```

## Helpers

### `optional`

Wrap any schema so it also accepts `undefined`.

```typescript
import * as p from '@vbudovski/paseri';

const schema = p.optional(p.string());
const result =
    schema.safeParse(undefined);
if (result.ok) {
    // result.value is of type `string | undefined`.
}
```

### `nullable`

Wrap any schema so it also accepts `null`.

```typescript
import * as p from '@vbudovski/paseri';

const schema = p.nullable(p.string());
const result = schema.safeParse(null);
if (result.ok) {
    // result.value is of type `string | null`.
}
```

### `chain`

Chain schemas to convert data types. `p.chain` takes the source schema as the first argument, the target schema as the
second argument, and a transformer function as the third argument.

:::caution
The transformer function must *not* throw! Instead, return an error code using the `err` helper function.
:::

```typescript
import * as p from '@vbudovski/paseri';

const schema = p.chain(
    p.string(),
    p.bigint(),
    (value) => {
        try {
            return p.ok(BigInt(value));
        } catch {
            return p.err(
                'invalid_bigint',
            );
        }
    },
);
// Number.MAX_SAFE_INTEGER + 1 as a string.
const result = schema.safeParse(
    '9007199254740992',
);
if (result.ok) {
    // result.value is of type `bigint`.
}
```

:::note
The error code `'invalid_bigint'` is a custom code. Custom error codes require a
[custom messages object](/reference/messages/#custom-error-codes) to work with `result.messages()`.
:::

## Types

### `Infer`

You can use the `Infer` type to determine the result type of any schema. This is useful, because you don't want to
manually define a type that could easily get out of sync with the schema definition.

```typescript
import * as p from '@vbudovski/paseri';

const schema = p.object({
    foo: p.string(),
    bar: p.number(),
});

type MySchema = p.Infer<typeof schema>;
// MySchema will be { foo: string, bar: number }.
```

## Standard Schema

Paseri also provides an interface that's compatible with [Standard Schema](https://standardschema.dev/schema). This
makes it possible to use Paseri with a number of tools and frameworks without having to write custom wrappers.
